# AutoGantt PRD（输入DDL自动生成可视化甘特图）

## 1. 需求背景

### 1.1 业务问题

多项目并行的生产研发中，排期通常散落在IM/表格/不同系统内，导致：

- 需求与环节节点不可视、不可追踪，无法形成跨项目的统一时间轴
- 需求变更频繁，手工重排成本高且易引入依赖冲突
- 同一项目的不同职能仅关注某一环节节点，缺少“按环节”的聚合视图
- DDL倒推时需考虑周末、节假日区间、调休工作日，手工计算易错
- 不同管线/项目存在前后置依赖，缺少直观的依赖展示与自动校验

### 1.2 产品目标

AutoGantt提供“开发范式 + 需求录入/导入 + 工作日历”三要素，自动生成可拖拽编辑的甘特图时间轴，并在关键节点自动提醒。

### 1.3 范围约束

- 页面结构：共4页，首页为“时间轴”，其余依次为“开发范式”“需求”“设置”
- 时间范围：日历支持选择到2026年及之后50年
- 排期规则：以工作日为单位计算（跳过周末/节假日；调休工作日视为工作日）

## 2. 用户画像

| 角色                      | 典型职责                   | 核心诉求                                         | 关键场景                   |
| ------------------------- | -------------------------- | ------------------------------------------------ | -------------------------- |
| 项目经理                  | 多项目排期、推进、风险管理 | 一键生成时间轴、拖拽调整、导出汇报、依赖冲突预警 | 日常看盘/周会汇报/排期变更 |
| 产品经理                  | 需求收集、优先级管理       | 快速录入/Excel批量导入、级别管理、变更可追踪     | 需求收集/评审/上线前排期   |

## 3. 核心业务流程

### 3.0 登录与存储选择（全局前置）

用户首次进入产品时，默认以“本地模式（无需登录）”运行；用户可在任意时刻登录并开启云端同步。

1. 未登录：
   - 所有数据写入本地存储
   - 可完整使用排期、导入导出、甘特图拖拽等能力
   - 仅“管理员级配置”（节假日全局配置、字段定义、权限）在本地生效
2. 登录后：
   - 用户可选择“保持本地 + 开启云端同步”或“仅云端（本地缓存）”
   - 管理员权限按账号角色生效（见6.5）
   - 支持多设备访问同一数据集

### 3.1 管理员初始化（设置页）

1. 配置全局工作日历：周末规则（默认双休）、节假日区间、调休工作日区间
2. 配置下拉项：模板分类、需求级别、管线（Pipeline）
3. 配置字段：
   - 范式模板字段（影响“开发范式”页新建/编辑表单）
   - 需求字段（影响“需求”页表单与导入模板）

### 3.2 录入开发范式模板（开发范式页）

用户操作场景：研发负责人创建一套”生产项目开发范式”。

1. 在”模板分类”下拉中新增分类（如”增长/基础设施”），或删除不再使用分类
2. 新建范式模板：填写模板名称、分类、适用管线（可选）
3. 维护环节列表：为固定类别下的各环节填写参考人天（单位：人天），并标记可选DDL节点（里程碑）
4. 配置环节依赖：为每个环节配置开始条件（见4.3）
5. 保存模板；模板可被需求引用
6. **Excel批量导入范式（迁移场景）**：
   - 下载标准Excel模板 → 按列填写各范式的环节信息 → 上传 → 系统批量创建范式模板
   - 模板格式对齐现有甘特图惯用格式，方便从飞书表格/Excel直接迁移已有排期范式

### 3.3 录入/导入需求（需求页）

用户操作场景：产品经理收集本月需求并批量导入。

1. 单条录入：填写需求名称、级别、数量、预期交付时间等
2. Excel导入：
   - 下载模板 → 填写 → 上传
   - 系统预校验（必填、日期格式、下拉值合法）→ 支持“部分导入”与错误定位
3. 对需求做增/减/变更：修改字段、逻辑删除/恢复（见异常逻辑）
4. 为需求选择：
   - 关联范式模板
   - 所属管线
   - 计划排期输入方式：输入项目总DDL（倒推）或输入开始日期（正推）
   - 跨项目依赖（可选）：依赖其他项目/需求的某一环节完成

### 3.4 查看与管理时间轴（时间轴页：首页）

用户操作场景：项目经理在周会上用周视图汇报。

1. 进入首页“时间轴”，系统合并展示：需求页手动录入 + Excel导入的需求
2. 选择视图：日/周/月（年切换作为范围切换能力；见4.2.2）
3. 查看甘特图：按“项目-需求-环节”结构展示；跨管线依赖以连线表示
4. 按需筛选：仅看某一环节（方便职能负责人看盘）、或按管线/级别筛选
5. 导出Excel：导出结果与当前周视图基本一致（见4.2.7）

### 3.5 拖拽调整排期（时间轴页）

用户操作场景：开发延期，项目经理拖拽后自动重排后续环节。

1. 在甘特图直接拖拽环节条：
   - 拖拽中间：整体平移（起止日期同时变化）
   - 拖拽左边缘：调整开始日期
   - 拖拽右边缘：调整结束日期
2. 松手时系统校验：
   - 不得落在非工作日（自动吸附到最近工作日并提示）
   - 不得违反依赖（否则提示冲突与建议处理方式）
3. 校验通过后自动重算后置依赖环节的起止时间
4. 用户确认是否保存（保存后对该需求生成变更记录，并触发通知）

## 4. 功能模块拆解

### 4.1 信息架构

- 全局Header导航：时间轴（首页）→ 开发范式 → 需求 → 设置
- Header右上角：未登录显示“登录/注册”；已登录显示用户菜单（账号、同步状态、退出）

### 4.2 时间轴（首页）

#### 4.2.1 展示结构

- 左侧列表：项目/需求树
  - 右侧收起按钮：点击项目名称左侧的折叠/展开，与甘特图行同步
  - 宽度调节：左侧树宽度可拖拽调整，最小60px
- 右侧画布：甘特图
  - 环节条：颜色按环节分类/自定义色
  - 依赖连线：展示前后置依赖；跨管线依赖需清晰可辨

#### 4.2.2 时间范围与视图

- 视图类型：日/周/月/年
- 年份切换：
  - header可水平滑动查看当前年份的完整日/周/月刻度
  - 切换到下一年需显式切换（按钮或年份选择器）
- 可选范围：支持到2026年及之后50年
- 周末显示：仅日视图将周末格标灰；周/月/年视图可不区分
- 周视图宽度：支持用户调整，最小40px
- Header刻度格式：日/周视图使用MM-DD，月视图使用YYYY-MM，年视图使用yyyy
- 网格线：日/周/月刻度需以更淡的纵向线贯穿Header与画布，便于对齐阅读

#### 4.2.3 自动生成规则（由范式驱动）

- 输入：需求（含数量、级别、预期上线时间/项目DDL） + 关联范式模板 + 工作日历
- 输出：按范式环节顺序生成每个环节的开始/结束日期与节点（里程碑）
- 计算口径：以“工作日”计算人天与持续时长；跳过节假日区间；调休工作日参与计算

#### 4.2.4 拖拽交互（必须支持）

- 拖动环节条中间：左右移动（起止同时平移）
- 拖动左侧边缘：调整开始日期
- 拖动右侧边缘：调整结束日期
- 拖拽后影响：
  - 后续依赖环节自动重新计算
  - 若存在跨项目依赖，被依赖方变更需触发依赖方冲突检测与提醒

#### 4.2.5 筛选与聚合

- 按环节筛选：仅展示所有需求的同一环节（满足职能看盘）
- 按管线筛选：聚焦某条管线的所有项目
- 按需求级别筛选：P0\~Pn（可配置）

#### 4.2.6 依赖展示（跨项目/跨管线）

- 支持在时间轴上直观看到：某管线项目完成某环节后，其他管线项目才能开始其某环节
- 交互要求：点击连线/依赖标识可查看依赖详情（来源环节、目标环节、触发条件）

#### 4.2.7 导出Excel（周视图对齐）

- 导出基准：以当前“周视图”为主
- Excel结构：
  - 表头：周刻度（与时间轴header一致）
  - 左列：自上而下为项目与其下的需求（与左侧树一致）
  - 环节块：有不同颜色，落在其起止日期对应的周格线内
- 导出一致性：导出结果在视觉上与页面周视图“基本一致”（同粒度、同顺序、同颜色映射）

#### 4.2.8 提醒与通知

- 节点提醒：到达里程碑节点/环节截止日前，按配置触发提醒
- 变更通知：被依赖环节变更时，通知所有依赖方（至少站内通知；渠道可配置为后续迭代项）

### 4.3 开发范式（页面2）

#### 4.3.1 模板与分类

- 模板分类下拉：用户可自行新增/删除分类
- 模板管理：增/删/改/查；模板被引用时的删除策略见异常逻辑

#### 4.3.2 环节配置（固定类别 + 可配置字段）

- 环节必须属于某一固定类别（类别集合可在设置维护）
- 每个环节需配置参考人天（整数）
- 可标记为DDL节点（里程碑）

#### 4.3.3 依赖关系（开始条件）

对每个环节可配置一个或多个前置依赖，支持：

- 串行（FS，前置100%完成后开始）
- 百分比重叠（FS + 完成百分比X%，前置完成X%后开始）
- 固定偏移（FS + X天，前置完成后X个工作日开始）
- 固定偏移（SS + X天，前置开始后X个工作日开始）
- 多依赖（AND，多个条件全部满足后开始；开始时间取各条件约束的最大值）

#### 4.3.4 Excel批量导入范式

**设计目的**：支持将飞书表格/Excel中已有的甘特图排期范式一键迁移至系统，降低初始化成本。

**Excel模板列结构（固定列顺序）：**

| 列  | 字段         | 必填 | 说明                                                                                                                 |
| --- | ------------ | ---- | -------------------------------------------------------------------------------------------------------------------- |
| A   | 范式名称     | 是   | 同一范式的所有环节行填写相同名称；系统按此字段分组创建模板                                                           |
| B   | 环节名称     | 是   | 该环节的名称，如"需求设计""开发实现"                                                                                 |
| C   | 参考人天     | 是   | 整数，该环节的标准工期（人天）                                                                                       |
| D   | 前置依赖环节 | 否   | 填写前置环节的名称（与B列值对应）；多个依赖用英文逗号分隔                                                            |
| E   | 依赖偏移天数 | 否   | 整数；配合F列使用，表示偏移天数（finish_offset_days / start_offset_days）                                            |
| F   | 依赖关系类型 | 否   | 下拉选项：`FS`（完成后开始，默认）/ `SS`（开始后开始）；留空默认 `FS`                                                |
| G   | 触发方式     | 否   | 下拉选项：`finish_100`（默认）/ `finish_percent` / `finish_offset_days` / `start_offset_days`；留空默认 `finish_100` |
| H   | 里程碑节点   | 否   | 下拉选项：`L0` / `L0.5` / `L1` / `L2`；留空表示非里程碑节点                                                          |
| I   | 模板分类     | 否   | 下拉选项：对应系统中已有分类；留空默认归入"通用"                                                                     |

**导入规则：**

- 按 A 列"范式名称"分组，同名的连续行合并为一个 `ParadigmTemplate`，环节按行顺序排列
- D 列依赖填写环节名称（而非 ID），系统在同一范式内按名称匹配 `preStageId`
- 系统预校验后支持"部分导入"：通过校验的范式整体导入，失败的范式整体跳过并定位到首个错误行
- 导入成功后范式出现在左侧列表，可继续手动编辑

### 4.4 需求（页面3）

- 表单模式收集需求，支持增/减/变更
- Excel批量导入：支持映射需求字段、错误行定位、部分导入
- 需求级别下拉：用户可自行新增/删除级别
- 与排期关联：选择范式模板、管线、工作日历，并输入项目DDL/开始日期
- 跨项目依赖：可指定依赖“其他项目/需求的某环节”

### 4.5 设置（页面4）

#### 4.5.1 工作日历

- 节假日：支持按区间维护（开始日期/结束日期），不可仅标注单天
- 调休：支持将某些周末标记为工作日（按区间/单日均可落地为区间）
- 生效规则：默认全局；后续可扩展到“管线/项目自定义日历”

#### 4.5.2 字段与下拉项

- 字段增删：在设置中增减或删除字段，会直接反映到“开发范式”页新建模板表单
- 下拉项维护：模板分类、需求级别、管线

#### 4.5.3 账号与同步

- 登录方式：邮箱验证码/魔法链接（MVP），可扩展企业SSO（后续）
- 存储模式：
  - 本地模式：仅本地保存，不要求登录
  - 云端模式：登录后保存到云端（本地仅缓存）
  - 本地+云端同步：登录后开启双端一致性
- 同步控制：手动“立即同步”、自动同步开关、同步失败重试
- 同步状态：
  - 已同步/同步中/离线待同步/有冲突需处理
- 数据管理：
  - 导出本地备份（JSON/Excel）
  - 清空本地缓存（不影响云端数据）

## 5. 字段说明

说明：以下为核心字段（不含审计字段如`createdAt/updatedAt`）。字段在“设置”中可扩展，但核心计算字段为强制字段。

### 5.1 工作日历

| 字段        | 类型 | 必填 | 说明                                          |
| ----------- | ---- | ---- | --------------------------------------------- |
| `name`      | 文本 | 是   | 名称，如“国庆假期”/“调休上班”                 |
| `startDate` | 日期 | 是   | 区间开始（含）                                |
| `endDate`   | 日期 | 是   | 区间结束（含）                                |
| `type`      | 枚举 | 是   | `holiday`（非工作日）/`workday`（调休工作日） |

### 5.2 范式模板（开发范式）

| 字段             | 类型 | 必填 | 说明               |
| ---------------- | ---- | ---- | ------------------ |
| `templateName`   | 文本 | 是   | 范式名称           |
| `categoryId`     | 下拉 | 是   | 模板分类（可维护） |
| `stageTemplates` | 数组 | 是   | 环节定义列表       |

环节定义（StageTemplate）：

| 字段                  | 类型 | 必填 | 说明                                                    |
| --------------------- | ---- | ---- | ------------------------------------------------------- |
| `stageName`           | 文本 | 是   | 环节名称                                                |
| `stageCategory`       | 下拉 | 是   | 固定类别（可在设置维护类别集合）                        |
| `referencePersonDays` | 整数 | 是   | 参考人天（用于排期计算）                                |
| `isMilestone`         | 枚举 | 否   | 里程碑节点类型：`L0`/`L0.5`/`L1`/`L2`；空值表示非里程碑 |
| `dependencies`        | 数组 | 否   | 前置依赖规则                                            |

依赖规则（DependencyRule）：

| 字段         | 类型 | 必填 | 说明                                                                   |
| ------------ | ---- | ---- | ---------------------------------------------------------------------- |
| `preStageId` | 引用 | 是   | 前置环节                                                               |
| `relation`   | 枚举 | 是   | `FS`/`SS`                                                              |
| `trigger`    | 枚举 | 是   | `finish_100`/`finish_percent`/`finish_offset_days`/`start_offset_days` |
| `value`      | 数字 | 否   | `finish_percent`=0\~100；offset=工作日天数                             |

### 5.3 需求（需求页）

| 字段                 | 类型 | 必填 | 说明                                     |
| -------------------- | ---- | ---- | ---------------------------------------- |
| `requirementName`    | 文本 | 是   | 需求名称                                 |
| `levelId`            | 下拉 | 是   | 需求级别（可维护）                       |
| `quantity`           | 整数 | 是   | 数量（默认1）                            |
| `expectedLaunchDate` | 日期 | 是   | 预期上线日期（用于计划管理与校验）       |
| `pipelineId`         | 下拉 | 是   | 所属管线                                 |
| `templateId`         | 下拉 | 是   | 关联范式模板                             |
| `scheduleMode`       | 枚举 | 是   | `backward_from_ddl`/`forward_from_start` |
| `projectDDL`         | 日期 | 条件 | 倒推模式必填                             |
| `projectStartDate`   | 日期 | 条件 | 正推模式必填                             |

### 5.4 排期实例（时间轴渲染）

| 字段             | 类型 | 必填 | 说明                   |
| ---------------- | ---- | ---- | ---------------------- |
| `requirementId`  | 引用 | 是   | 归属需求               |
| `stageId`        | 引用 | 是   | 归属环节模板           |
| `startDate`      | 日期 | 是   | 环节开始（工作日对齐） |
| `endDate`        | 日期 | 是   | 环节结束（工作日对齐） |
| `manualOverride` | 布尔 | 是   | 是否被用户拖拽/编辑过  |

### 5.5 跨项目依赖

| 字段                  | 类型             | 必填 | 说明                               |
| --------------------- | ---------------- | ---- | ---------------------------------- |
| `fromStageInstanceId` | 引用             | 是   | 被依赖环节                         |
| `toStageInstanceId`   | 引用             | 是   | 依赖方环节                         |
| `trigger`             | 同DependencyRule | 是   | 触发条件（完成100%/完成X%/偏移等） |

### 5.6 数据存储规则（通用）

- 数据归属：
  - 本地模式：数据仅存在当前浏览器（同浏览器不同设备不共享）
  - 云端/同步模式：数据归属登录账号，按租户隔离（同账号多端共享）
- 生命周期：
  - 本地数据可能因清理浏览器缓存丢失，产品需提供“本地备份导出”能力
  - 云端数据支持“账号注销/数据删除”入口（管理员可见）
- 软删除：需求/项目/模板默认软删除（可恢复），避免误操作导致排期丢失
- 版本控制：所有可同步记录包含`version`与`updated_at`，用于增量同步与冲突检测
- 敏感信息：不存储明文密码；登录凭证由认证服务托管；前端不得在日志输出Token

## 6. 异常逻辑

### 6.1 时间计算与工作日历

- `startDate > endDate`：阻止保存，提示“结束日期必须晚于开始日期”
- 落在非工作日：
  - 自动吸附到最近工作日（规则：向后吸附为默认；用户可在弹窗中选择向前）
  - 记录一次自动修正提示
- 计算超出时间范围（2026年后50年）：阻止并提示“超出系统支持范围”

### 6.2 依赖冲突

- 创建/编辑依赖时检测循环依赖：阻止并提示冲突链路
- 拖拽导致依赖不满足：
  - 方案A：阻止保存并高亮冲突依赖
  - 方案B：允许保存但要求选择“自动顺延后续环节/仅保存当前环节”（默认推荐A，避免产生不可执行计划）
- 被依赖方变更：自动触发依赖方重算；若重算后超出其DDL/上线日期，标记为风险并通知

### 6.3 Excel导入

- 模板不匹配/列缺失：阻止导入，提示缺失列清单
- 日期格式非法：定位到行列并给出示例格式
- 下拉值不合法（级别/管线/分类）：提示可选值；支持”自动新增下拉项（需管理员权限）”或”按错误跳过”
- 重复数据：按（需求名称+管线+预期上线日）做默认去重策略，提供”覆盖/跳过/新增”选项

#### 6.3.1 范式Excel导入额外校验

- **A列范式名称**为空：跳过该行，记录警告
- **B列环节名称**为空：该范式整体校验失败，整组跳过并提示行号
- **C列参考人天**非正整数：该范式整体校验失败
- **D列依赖环节名称**在同一范式内找不到对应环节：该范式整体校验失败，提示找不到的依赖名称
- **G列触发方式**与**F列关系类型**组合不合法（如 `SS` + `finish_percent`）：记录警告，自动降级为 `SS` + `finish_100`
- **同一范式内环节名称重复**：校验失败，提示重复行号

### 6.4 字段与下拉项变更

- 删除被引用的下拉项：不允许直接删除；需先迁移存量数据或标记为“停用”
- 删除字段：
  - 对历史数据保留但不再展示（避免数据丢失）
  - 导入模板同步更新

### 6.5 权限

- 工作日历、字段与下拉项维护：仅管理员可操作
- 需求与排期编辑：按项目/管线授权（若未配置，默认所有人可编辑，作为MVP策略）

### 6.6 登录与会话

- 未登录访问：允许访问核心功能；若进入“需要管理员权限”的能力，提示登录
- 会话过期：
  - 云端读写返回401时，弹窗提示重新登录
  - 本地模式不受影响，允许继续编辑并进入“离线待同步”状态
- 退出登录：
  - 若本地存在未同步变更，退出前提示“先同步/继续退出（变更仅保留本地）”

### 6.7 本地/云端数据一致性

- 离线编辑：
  - 本地写入成功即视为成功；显示“离线待同步”标识
  - 网络恢复后自动发起同步
- 同步冲突（同一记录被多端修改）：
  - 默认策略：字段级合并（不冲突字段自动合并）
  - 冲突字段：弹窗展示“本地版本 vs 云端版本”，用户选择其一
  - 冲突处理完成前，该记录在时间轴上以“⚠ 待处理”提示
- 禁止隐式覆盖：任何会造成跨端数据丢失的覆盖操作必须二次确认

### 6.8 本地存储异常

- 浏览器存储空间不足/写入失败：
  - 阻止继续编辑并提示“空间不足，请导出备份后清理缓存或切换云端模式”
  - 提供快捷入口：导出备份、前往账号与同步切换云端

## 7. 验收标准

### 7.1 核心场景验收（必须通过）

- 范式模板：可新增/编辑/删除模板；模板分类下拉可新增/删除并即时生效
- 需求录入：支持单条录入与Excel批量导入；导入支持错误行定位与部分导入
- 自动排期：输入项目DDL后，系统按工作日历从DDL倒推生成各环节起止日期
- 工作日历：节假日按区间生效；调休工作日可覆盖默认周末规则；日视图周末格标灰
- 时间轴视图：支持日/周/月/年切换；年份可切换；周视图单元宽度可调且最小40px
- 拖拽编辑：支持三种拖拽方式；拖拽后后置依赖环节自动重算；保存前必须校验依赖
- 环节筛选：可只看所有需求的同一环节，并保持左侧树与画布行对齐
- Excel导出：导出周视图结果，表头为周，左列为项目/需求，环节块颜色与时间位置对齐
- 登录：支持邮箱验证码/魔法链接登录；登录后可正常读写云端数据
- 存储模式：可切换本地/云端/本地+云端同步；切换后数据归属与读写行为符合4.5.3
- 离线可用：断网情况下可编辑（需求、拖拽排期、节假日本地修改），恢复网络后可同步
- 冲突处理：多端同时修改同一需求/环节时，能触发冲突提示并完成手动决策
- 本地备份：可导出本地备份并在清空缓存后通过导入恢复核心数据

### 7.2 性能与容量

- 周视图渲染：50个项目、200个需求、总计2000个环节实例，滚动与拖拽无明显卡顿
- 导入：1000条需求在10秒内完成校验与入库（本地/网络环境按实现定义）
- 拖拽：交互响应<200ms；松手重算<800ms

### 7.3 可用性与一致性

- 关键操作（删除需求/保存拖拽结果/字段变更）均有二次确认
- 所有错误提示包含：错误原因 + 定位信息（行列/字段）+ 可操作的修复建议

## 8. 工程规范与变更管理（强制）

### 8.1 文档与实现一致性

- 单一事实来源：本文档与技术架构、UI设计文档视为对外契约
- 变更规则：开发过程中如需求提示与现有设计不一致，允许调整实现，但必须同步更新相应文档，确保“文档=代码=功能”一致

### 8.2 代码可维护性（注释与可读性）

- 代码注释：生成代码时，文件头、函数/方法、条件判断、循环逻辑必须添加JSDoc，说明参数、返回值与逻辑目的
- 配置注释：生成配置、schema时，尽可能为每行配置添加注释，说明作用与可选项
- 数据库注释：生成关系型数据库表结构时，建表DDL必须为表/字段添加中文COMMENT

### 8.3 安全与最小改动原则

- 最小化改动：仅修改实现需求必需的代码，不做冗余改动
- 密钥与凭证：严禁将Token/密钥写入代码仓库；严禁在日志中输出敏感信息

### 8.4 需先确认的高风险操作

若非需求提示词主动要求，涉及以下操作必须先确认并获得同意后再执行：

- 变更数据库结构（新增表、修改字段、删除表）
- 调整接口结构（请求参数、响应格式、接口路径）
- 清除数据表数据（无论数据量大小）

### 8.5 临时脚本与仓库卫生

- 开发/调试/测试产生的临时工具脚本、测试数据、调试log等，必须加入`.gitignore`，避免提交进repo

### 8.6 代码提交规范（强制）

- 会话结束：必须升级`package.json`版本并提交代码
- 提交前质量门禁：必须执行ESLint扫描 + Prettier扫描 + TypeScript检查
- 变更日志：提交前更新`history.md`，按模板追加`#版本（#日期) #一句话简要说明改动内容`
- 提交信息：使用Conventional Commits规范编写git commit message
- 提交前检查：核对文件列表，非源代码文件需加入`.gitignore`
